## 布局 Bug 修复

**问题**：右侧画布区域（canvas-area）完全不显示，左侧工具栏占据整个窗口宽度。

**根因分析**：
1. WKWebView 在 SwiftUI NSViewRepresentable 中默认 intrinsicContentSize 不为 .zero，导致 auto-layout 计算异常
2. CSS 中 body 使用 `height: 100vh` 在 WKWebView 中可能不准确，应改用 `height: 100%` 配合 html `height: 100%`
3. toolbar 缺少 max-width/min-width 约束，flex 布局下可能被意外拉伸

**修复措施**：
1. **WebView.swift**：覆盖 `intrinsicContentSize` 返回 `.zero`，设置 `autoresizingMask = [.width, .height]`
2. **style.css**：
   - html/body 都加 `height: 100%; width: 100%`
   - `.app` 改为 `height: 100%; width: 100%; overflow: hidden`
   - `.toolbar` 加 `max-width: 260px; min-width: 260px`
   - `.canvas-area` 加 `min-width: 0`

## 修复导出图竖排文字上下颠倒

### 问题
导出图片中竖排文字每列字符顺序反转。例：李清照《声声慢》"寻寻觅觅"在导出图中显示为"觅觅寻寻"。

### 第一次尝试（CTM翻转）——失败
`exportImage()` 中对 CGContext 做 translate(0, height) + scale(1, -1) 翻转 y 轴。这纠正了列内字符顺序，但导致 `NSString.draw(at:)` 渲染的文字本体上下颠倒——因为 CTM 翻转影响了对文本的渲染，而不仅仅是定位。

### 第二次修复（isFlipped参数）
不修改 CTM，改为让 CanvasRenderer 感知坐标系方向。
- `CanvasRenderer.draw()` 新增 `isFlipped: Bool` 参数（默认 true）
- `drawVerticalText()`: `isFlipped=false` 时，y 坐标计算改为 `center + totalHeight/2 - rowIdx * spacing`，使首字符在最上方（大 y）
- `drawHorizontalText()`: `isFlipped=false` 时，startY 从顶部（大 y）向下递减
- `exportImage()` 传入 `isFlipped: false`，移除 CTM 翻转代码

此修复同时解决竖排/横排文字的导出顺序问题，且不影响屏幕渲染（默认 isFlipped=true）。
